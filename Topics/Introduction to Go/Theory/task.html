Introduction to Go

A major tool of computer science is programming languages. We use them to develop computer programs. In the modern world, there is a wide variety of programming languages, every one with its own community of fans. Let's find out what makes Go special and what's there to like about this language.

Go programming language?

Just a very brief history note, if you're curious:

In 2007, Google began developing the Go programming language as an internal project. Two Unix luminaries, Rob Pike and Ken Thompson, designed the original. Two years later, on November 10, 2009, the software was released under a liberal open-source license. A version of Go 1.0 that was ready for production was released in March 2012. There have been three versions of Go since 2012, and a Go 2.0 version is in the works! Google's team of designers works on the project along with Russ Cox, Andrew Gerrand, Ian Lance Taylor, and others. And thanks to all these people, you've got the chance to learn this programming language.

Go is usually called Golang, the same name as the Go programming language. We will use them interchangeably in this topic.

Advantages of Go

Go is a pragmatic language: its main purpose is to solve real-world problems. You can see it in the below examples:

  • The original intent of Go was to be used in the server-centric world of web servers, storage architecture, due to the fact that Go is compiled to the native code. Go programs usually execute very fast and have a low memory footprint. That's why you can use this language as a system programming language.
  • Golang excels at handling massive concurrency and making event-processing easy. That makes it a good choice for game servers and IoT solutions. Another useful link to learn about successful examples in Go is the Github wiki.
  • And finally, you can use Go as a general programming language. It is a great tool for developing scripts and solving text-processing problems.

The language supports a range of programming paradigms, including imperative programming, object-oriented programming, procedure, and functional programming. The Go FAQ contains an explanation of these designs by the Go team.

Disadvantages of Go

While Go has many advantages over other programming languages, it still has some disadvantages you should be aware of when learning it:

  • Go's standard library lacks many basic functions (for example, finding the biggest number in a slice of numbers), so you have to implement them by hand. Besides, the Go libraries don't support GUI programming by default, which is the advantage of the Java standard library.
  • As a result, Go's standalone binaries are large, because they require the embed garbage collector and the entire Go runtime as well. The "Hello World" program might consume 2MB in file size.
First Go example

Let's look at the first example of Go's code. It prints the "Hello Golang" string to the terminal (suppose this example is contained within the main.go file):

package main

import "fmt"

func main() {
    // Print "Hello Golang" string to the screen
    fmt.Println("Hello Golang")
}

When we write comments in Go, we use the C++ style: // for one-line comments that we usually place at the end of the code line, and /* ... */ for multi-line comments. The Go programming language is organized into packages, and every Go program should have the main package containing the main() function, which serves as the entry point for the program (the function that is executed first).

Now click the run button in JetBrains' Goland (or Intellij IDEA) or run the program directly in the terminal by executing the go run main.go command.

Congrats! You've just used Go for the first time.
